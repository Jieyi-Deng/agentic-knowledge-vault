# Information Stories

## Summary

An information story is a user-story-like design tool for portable information structures. Instead of starting from whatever data is available, it starts from a user, an information need, and an intended action or outcome.

## Template

A common form is:

> As a [person / citizen / business / role], I need access to [specific information] so that I can [achieve a goal].

The course uses this to keep information architecture focused on use rather than on technology alone.

## Why It Matters

Information stories act as boundary objects. They help collaborators build shared understanding around:

- who the user is
- what information is needed
- what context gives the information meaning
- what structure can support the action
- which fields, metadata, and access methods are necessary

## Examples From Class

- A community member needs public transportation information so they can move around the city.
- A citizen needs school funding information so they can understand public budgeting.
- A person needs access to benefits or social security information so they can claim rights.
- A student needs building access information integrated with enrollment and location data so a keycard can permit entry to the right class location.

## Connection to Product Development

Information stories align with new product development because they foreground need validation. The class emphasized that product work should not start only from available data. It should begin with the user’s goal, then identify which information structures and transformations can support that goal.
